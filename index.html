🍽️ Feastie - AI-Powered Restaurant Reservation Agent

Ever wished you had a personal assistant for restaurant bookings? Meet Feastie! Built an intelligent reservation agent powered by Qwen 2.5-7B that understands natural language and handles end-to-end bookings like a pro.


The coolest part? When your preferred time slot is taken, Feastie doesn't just say "sorry, unavailable" - it actually suggests better alternatives based on availability, location, and timing. This smart recommendation system boosted successful bookings by ~30% in testing!


Built the entire system using a small language model (because not everything needs GPT-4!), SQLite for persistence, and designed intelligent fallback logic that makes the agent feel genuinely helpful, not robotic.

The Challenge

Traditional booking systems require multiple clicks, form fills, and lack intelligent suggestions when preferred slots are unavailable. Users often abandon the process out of frustration.

My Approach

Leveraged Qwen 2.5-7B (a lightweight but powerful SLM) to create a conversational agent that interprets natural language requests. Designed a smart recommendation engine that proactively suggests alternatives, and built the entire system with reliability and user experience as top priorities.

Results & Impact

✅ 30% increase in successful booking completion
✅ Natural language understanding with high precision
✅ Handles create, modify, and cancel operations autonomously
✅ Intelligent fallback suggestions for unavailable slots

Key Learnings

Discovered that smaller, fine-tuned models can outperform larger models for specific tasks. Learned the importance of building fallback logic that feels helpful, not mechanical - the difference between "slot unavailable" and "how about 7 PM instead, which is just as popular?"

🌱 Forget-me-Knot - Smart Plant Care Telegram Bot

Built because I kept killing my plants (oops!). This Telegram bot is like having a green-thumbed friend who never forgets to remind you about watering schedules - works seamlessly across group chats and tracks individual users.


The technical magic? Architected a lightweight yet powerful backend on Vercel with serverless functions, Redis for persistent state management, and GitHub Actions for automated reminder workflows. The result is a bot that's always on, always reliable, and handles multiple users effortlessly.


What I love about this project: It solves a real problem (keeping plants alive!), showcases modern serverless architecture, and demonstrates how to build production-ready bots with proper state management and security.

The Challenge

Plant owners need consistent reminders but existing solutions lack multi-user support, group chat integration, and reliable scheduling. Plus, nobody wants to host a server just for a plant reminder bot!

My Approach

Built a serverless-first architecture using Vercel for API routing and Redis for durable state management. Implemented GitHub Actions for scheduled reminders, ensuring the bot runs reliably without a dedicated server. Designed the system to handle multiple users and group chats with individual tracking.

Results & Impact

✅ Zero-maintenance serverless deployment
✅ Multi-user support with per-user history tracking
✅ Group chat functionality for shared plant care
✅ Automated reminder workflows with 99.9% uptime
✅ Secure secret management for API keys

Key Learnings

Mastered serverless architecture patterns and learned how to combine different cloud services (Vercel, Redis, GitHub Actions) into a cohesive system. Discovered that great DevOps isn't about complexity - it's about choosing the right tools that work reliably together.

Python Telegram Bot API Vercel Redis GitHub Actions Serverless
